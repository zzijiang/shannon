RNN 的问题：
记忆力太差
上一节我们学了 RNN——它有隐藏状态，能处理序列数据。

但它有个致命问题。试试读这段话：
"我从小在法国长大， 在那里度过了整个童年，上了当地的学校，交了很多朋友， 后来去了巴黎读大学，毕业后在一家科技公司工作了三年， 然后去了美国硅谷做了两年工程师，又回到欧洲…… 所以我的母语是 ______"
你知道答案是法语，因为你记住了开头的"法国"。

但 RNN 读到最后时，"法国"的信息已经几乎消失了——
每一步都在乘以小于 1 的权重，经过几十步就约等于零。
02 / 07
LSTM 的解决方案：
三道门
LSTM（Long Short-Term Memory）加了一条记忆传送带——
信息可以在上面不衰减地一直传下去。

然后用三道门来控制什么信息上传送带、什么信息被移除：
🗑️
遗忘门
Forget Gate
决定丢掉旧记忆中的哪些部分
"法国"这个信息还有用吗？
📥
输入门
Input Gate
决定写入什么新信息
"美国"这个新信息要不要存？
📤
输出门
Output Gate
决定输出记忆中的哪些部分
回答"母语"时该取出什么？
每道门输出 0 到 1 之间的数：0 = 完全关闭，1 = 完全打开。
03 / 07
用生活场景理解
想象你在准备考试，面前有一张笔记纸（记忆单元 C）。
每学一章新内容，你会：
① 遗忘门：划掉笔记上已经过时的内容
（"这个公式考纲删了，划掉"）
② 输入门：把新学的重点写上去
（"这个定理很重要，记下来"）
③ 输出门：考试时，从笔记里挑相关的来回答
（"这道题问的是XXX，我看看笔记里有没有"）
笔记纸就是 LSTM 的记忆单元（Cell State）。
三道门控制了哪些信息留在纸上、哪些被擦掉、哪些被读出来。

05 / 07
LSTM vs RNN
现在你理解了两者的区别：
RNNLSTM
记忆机制 只有隐藏状态 h
每步乘以权重，会衰减		 隐藏状态 h + 记忆单元 C
C 通过传送带直传，不衰减
长距离依赖 差：几十步后遗忘 好：门控制信息保留
参数量 少（1组权重） 多（4组权重：3个门+候选记忆）
训练速度 快 慢（参数多，计算多）
适用场景 短序列、简单模式 长文本、语音、时间序列
💡 NOAI 复赛中：2024第3题（文本分类）建议用 LSTM，2025第1题（德语分词）用 BiLSTM 效果最好。
06 / 07
还有一个变种：GRU
GRU（Gated Recurrent Unit）是 LSTM 的简化版：
把 3 道门合并成 2 道（重置门 + 更新门），去掉了独立的记忆单元。
🏠
LSTM
3道门 + 记忆单元
功能最全，参数最多
长序列首选
🏕️
GRU
2道门，更简洁
效果接近 LSTM
训练更快，适合资源有限
🎒
RNN
没有门
最简单，但会遗忘
只适合短序列
NOAI 考试：知道 GRU 是 LSTM 的简化版就够了。PyTorch 中用 nn.GRU()，接口和 nn.LSTM() 几乎一样。
